# 翻譯小貓設計規格

## 目標

以 Read Frog 1.35.1 為基礎建立 Chrome Manifest V3 擴充功能「翻譯小貓」，保留免費翻譯與 API Key 供應商，新增網路／本機 PDF 的雙欄翻譯閱讀器，並讓 YouTube 字幕翻譯的啟動、錯誤與重試都能即時回應。

## 發布與授權

- 第一版供使用者本人使用，以 Chrome「載入未封裝項目」安裝。
- 保留 GPL-3.0、原作者版權與第三方聲明；衍生專案同樣以 GPL-3.0 發布原始碼。
- 專案建立於使用者已登入的 GitHub 帳號，預設為 private，避免未經確認直接公開。

## 架構

### 現有能力

保留 Read Frog 的 WXT／React 架構、內容腳本、背景服務、翻譯供應商設定、翻譯佇列、網頁雙語翻譯、選字工具列與 YouTube 字幕解析器。

### PDF 雙欄閱讀器

- 擴充功能辨識 `application/pdf`、`.pdf` 網址與 `file://` PDF。
- 使用者從 popup 點擊「開啟 PDF 翻譯本」後，在 extension page 中載入 PDF.js 閱讀器。
- 左欄渲染原 PDF；右欄依頁碼與文字區塊顯示翻譯本。
- 每個來源文字區塊有穩定的 segment id。翻譯區塊保留來源字元區間對應，允許任一側選字時，另一側對應區塊同步套用深色 highlight。
- 選取無法精確映射到單字時，退回句子或段落級同步標示，避免錯標。
- 網路 PDF 透過背景 fetch；本機 PDF 透過 `file://` 權限與使用者手動選檔兩條路徑支援。
- 翻譯按頁漸進執行，支援取消、重試、頁面快取與部分完成，不等待整份文件才顯示結果。

### YouTube 可靠性

- 支援設定「偵測字幕自動啟動」或播放器內常駐按鈕手動啟動，兩者可並存。
- 將啟動流程建模為 `idle → detecting → loading → translating → ready | error`。
- 監聽 YouTube SPA 導航、影片 id 變更、播放器節點重建與 Shorts reel 變更；任何變更都取消舊工作並重新掛載。
- 所有點擊立即反映狀態；逾時或字幕取得失敗顯示原因與「重試」，禁止無聲失效。
- 每個影片工作使用 generation token，舊影片的非同步回應不得覆蓋新影片狀態。

### 翻譯提示詞

- 免費翻譯供應商沿用既有請求格式。
- API Key／LLM 供應商分為 webpage、pdf、subtitle 三個提示詞場景。
- 共通要求：僅輸出譯文、保持語意與專有名詞一致、不添加解釋、不遺漏輸入、保留必要標記。
- PDF 提示詞要求維持段落／句子邊界與 segment id，避免摘要化。
- 字幕提示詞要求口語自然、簡潔、保持字幕切片順序，不將多行合併到無法對時。
- 所有模型輸出經結構驗證；缺失 segment 只重試缺失項，不重翻全部內容。

## 介面與品牌

- 名稱統一為「翻譯小貓」，預設介面為繁體中文。
- 視覺方向為溫暖紙張色、炭黑文字與橘棕重點色，保留高對比與鍵盤焦點。
- 使用 image2 產生原創小貓閱讀／翻譯主圖，再裁切為 Chrome icon 所需尺寸；不沿用 Read Frog 青蛙圖像。
- Popup 首屏顯示目前頁面類型、翻譯狀態與一個明確主動作；PDF 頁面顯示「開啟翻譯本」。

## 隱私與錯誤處理

- API Key 僅存在 Chrome storage，不寫入原始碼、log 或分析事件。
- 文件文字只送往使用者選定的翻譯供應商；介面清楚標示目前供應商。
- PDF fetch、解析、翻譯與 highlight 各自有可辨識錯誤訊息和重試入口。
- 無字幕、字幕 API 限流、影片切換、供應商逾時與取消不是靜默例外。

## 驗收

- `pnpm test`、`pnpm type-check`、`pnpm lint` 與 `pnpm build` 成功。
- 網路 PDF 與本機 PDF 均可進入雙欄閱讀器；頁面能漸進翻譯。
- 在來源或譯文選取單字／句子時，對側對應 segment 同步加深。
- YouTube 自動與手動模式都有即時狀態，換片後不殘留舊翻譯，失敗可重試。
- build 產物可由 Chrome 載入，名稱與 icon 均為翻譯小貓。
